# Prompt Bertahap Trae — Modul Bedah Dapil (Tanpa Database, CSV Langsung)

Jalankan prompt berurutan. Tunggu selesai per tahap.

**ARSITEKTUR:** Tidak pakai database/migration/model untuk data pemilu. CSV di-serve sebagai static file, JavaScript parse & render langsung di browser. Hanya perlu route, blade view, dan JS.

**DATA:** File CSV sudah ada di `storage/app/private/import/`:
- `tps_dprd.csv` (~970rb baris, separator `;`)
- `dpt_dapil1_rt_rw.csv`, `dpt_bekasi4_rt_rw.csv`, `dpt_dapil4_rt_rw.csv`, `dpt_dapil5_rt_rw.csv`
- Gambar peta: `dapil1.png`, `dapil4.png`

CORE FUNCTIONS YANG HARUS DIBUAT (logic sama dengan referensi):

1. parseSemicolonCsv(text) — parse teks CSV separator ; jadi array of objects (header row jadi key)
2. normalizeKey(value) — uppercase, trim, collapse whitespace
3. resolveLatestDapil(rawDapil, rawKecamatan) — fix mapping dapil: CIKARANG SELATAN → BEKASI 1
4. buildDataset(rows) — dari CSV TPS, bangun struktur: Map<dapil, { kecamatanMap, desaMap }>
   - Per desa: partyMap (partai → {partyVotes, candidateVotes, candidates})
   - Per TPS: partyMap
   - TPS 000 / nomor_urut=0 = suara partai, nomor_urut>0 = suara calon
5. buildDptDataset(rows) — dari CSV DPT, bangun Map<villageKey, { rwMap, totalDpt, male, female, gen_z, millennial, gen_x, boomer }>
6. analyzePks(partyMap, totalTps) — hitung ranking PKS, share, gap, lalu classifyPriority
7. classifyPriority({pksVotes, share, rank, gapShare}):
   - pksVotes <= 0 → ZONA BERAT
   - rank==1 && share>=0.3 → JAGA KUAT
   - rank==1 → AMANKAN
   - rank==2 && gapShare<=0.05 → REBUT REALISTIS
   - rank<=3 || share>=0.12 → GARAP INTENSIF
   - else → ZONA BERAT
8. buildScopeData(dapilObj) — filter villages by kecamatan/status/search, aggregate
9. render() — fungsi utama yang memanggil semua render sub-fungsi
